ConnectionEngine Architecture Sketch
@Observable
final class ConnectionEngine {
  private let profile: StudentProfileModel
  private var cancellables: Set<AnyCancellable> = []

  init(profile: StudentProfileModel) {
    self.profile = profile
    observeChanges()
  }

  private func observeChanges() {
    // When career path changes
    observe(\.careerPath) { [weak self] path in
      self?.updateCollegeFilters(for: path)     // ← College Discovery
      self?.regenerateActivitySuggestions()     // ← Activity System
      self?.updateAIContext()                   // ← AI Advisor
      self?.refreshScholarshipFilters()         // ← Scholarships
    }

    // When college is saved
    observe(\.savedCollegeIds) { [weak self] ids in
      self?.syncDeadlinesCalendar(ids)          // ← Already built ✓
      self?.generateApplicationRecords(ids)     // ← Applications
      self?.generateEssaySlots(ids)             // ← Essay Hub
      self?.updateDashboardUrgencyCard()        // ← Dashboard
    }

    // When grade changes or transcript uploaded
    observe(\.grade) { [weak self] grade in
      self?.updateGradeVisibility(grade)        // ← GradeFeatureManager
      self?.refreshRoadmapMilestones(grade)     // ← Roadmap
    }
  }
}

Critical Gaps — Ranked by Impact

🔴 #1 — No Backend (Supabase not wired)

AuthManager mocks state directly. StudentProfileModel lives in local SwiftData only. No user identity, no cloud persistence, no real data survives a reinstall.

Blocks: everything. Every single feature depends on real auth + persisted data.

🔴 #2 — ConnectionEngine doesn't exist

Career quiz result is stored but zero features read it. GPA is stored but AI doesn't know it. Saved colleges don't auto-generate application records. The app is 9 disconnected screens.

Blocks: personalization, auto-checklists, MATCH/REACH/SAFETY, activity suggestions.

🟠 #3 — AI is mocked with keyword matching

AdvisorChatViewModel hardcodes responses based on keyword detection. AIService.swift is correctly architected (streaming ready), but Bearer token auth is TODO and student context is never injected.

Blocks: the entire core value prop. AI counselor = most important feature.

🟠 #4 — Grade doesn't change anything

A freshman and a senior see identical UX. GradeFeatureManager doesn't exist. The app should be radically different at each grade — hiding application features from 9th graders, unlocking them for juniors/seniors.

Blocks: appropriate UX, grade-based feature locking, roadmap adaptivity.

🟡 #5 — Activity System not built

The 4 generals (athletics, volunteering, clubs, leadership) + 6 career-specific activities are the main differentiator from every other college app. Tracked per year. Rated 1-10. None of it exists yet.

Blocks: the core "start freshman year not senior year" pitch to users.

🟡 #6 — Transcript upload + AI parse missing

The whole class suggestion engine depends on reading past transcript grades. Right now GPA is a self-reported number in onboarding. Transcript upload → Gemini Vision parse → profile update is all missing.

Blocks: class suggestions, AP recommendations, AI academic context.

🟡 #7 — Post-acceptance checklist doesn't auto-generate

When application status flips to .accepted, the pre-app checklist should transform into a post-acceptance checklist (deposit, housing, immunization, official transcripts). This cascading transform isn't built.

Blocks: the second half of the product — what happens AFTER you get in.

🟡 #8 — College data is mock seed (not real API)

6,500 colleges are seeded as static mock JSON. The College Scorecard API key is configured but the actual API call is never made. Acceptance rates, tuition, SAT ranges could be stale or wrong.

Blocks: real MATCH/REACH/SAFETY, accurate chances calculator, live deadline data.
🎯 The Two Things That Unlock Everything
1. Wire Supabase (Sprint 1)

Real auth + real data persistence is the foundation. Add the SPM package, wire AuthManager to supabase.auth, sync StudentProfile to the DB, add Bearer token to AIService. Estimated: 1-2 days of focused work.

2. Build ConnectionEngine (Sprint 2)

The @Observable service that makes everything talk to everything. Start with the two highest-leverage cascades: careerPath → college filters + activities and savedCollegeIds → application records. Everything else branches from there.
